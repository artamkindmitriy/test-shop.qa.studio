## 🔍 4. ОБЪЕКТЫ И ФУНКЦИИ ТЕСТИРОВАНИЯ
### Объекты тестирования:
* **Frontend-часть** сайта `test-shop.qa.studio`.
* **Backend-часть** (публичные API-запросы, такие как эндпоинты для поиска, фильтрации, оплаты).

### Функции для тестирования:
* **Критический путь (Happy Path):** поиск товара ➡️ добавление в корзину ➡️ заполнение формы ➡️ успешное оформление заказа.
* **Математика корзины:** логика изменения количества товара кнопками `+` / `-` и автоматическое обновление итоговой суммы без перезагрузки.
* **Валидация полей:** проверка граничных значений, например в форме оформления заказа или в корзине.
* **Проверка адаптивной верстки сайта.**

## 🚦 6. КРИТЕРИИ И ОКРУЖЕНИЕ

### Критерии тестирования
* **Прохождение / Непрохождение:** Тест считается пройденным (Passed), если фактический результат совпадает с ожидаемым. Любое отклонение (ошибка верстки, код ответа 500 вместо 200, зависание формы) фиксируется как провал (Failed).
* **Приостановка:** Блокирующий дефект (Blocker) в самом начале Happy Path (например, невозможно добавить товар в корзину).
* **Завершенность:** Выполнение 100% ручных Smoke-тестов; написаны и стабильно запускаются запланированные автотесты; в Allure-отчете зафиксировано 100% прохождения; отсутствуют нерешенные дефекты уровней Blocker и Critical.

### Требования к окружению и данным
* **ПО:** ОС Windows 10/11, macOS, Android, iOS; браузер Google Chrome; Python 3.10+, PyCharm.
* **Тестовые данные:** Валидные данные для обязательных полей (под знаком `*`); невалидные наборы данных для проверки ограничений (спецсимволы, отрицательные числа, строки максимальной длины).
